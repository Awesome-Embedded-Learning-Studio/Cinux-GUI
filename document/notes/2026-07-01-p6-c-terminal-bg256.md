# P6-c — 终端 ANSI bg + 256 色 + 光标块

> P6 第三批。P5-e 的 SGR 只有 fg 16 色，补 bg（40-47）+ 256 色（38;5;N / 48;5;N）+ 光标块（visible cursor）。vim/less/彩色 prompt 体验更完整。

## 背景

P5-e SGR fg 16 色。`ls --color` 出彩色了，但无 bg（无高亮）/ 无 256 色（许多程序用）/ 无光标（不知输入位置）。P6-c 补齐。

## 目标

cells 加 per-cell bg + SGR 扩 bg/256 + 256 palette + paint per-cell bg + cursor block。Host ABI 零改动。

## 设计

- **cells 加 `bg_colors_[kMaxCols*kMaxRows]` + `cur_bg_`**（P5-e 只有 fg_colors_）。put_char_ 写 fg+bg；scroll 移 fg+bg；bg_at()。
- **`apply_sgr_` 重写**：P5-e 逐 code 回调，P6-c 改 **codes[16] 数组**遍历，可前瞻（38;5;N 需看下俩 code）。支持：
  - 0/39 → fg=白 bg=黑（reset）；49 → bg=default
  - 30-37/90-97 → fg（normal/bright）
  - 40-47/100-107 → bg（P6-c 新）
  - 38;5;N → fg=256；48;5;N → bg=256（P6-c 新，`i+=2` 跳过 5;N）
- **`palette_color(idx)`**（匿名 ns）：0-15 = kAnsiPalette[16]；16-231 = 6×6×6 立方（每通道 0 或 55+40×v）；232-255 = 灰阶（8+(idx-232)×10）。
- **paint per-cell bg + 256 fg + cursor block**：
  - 整 terminal fill 默认 bg（黑）。
  - 每 cell：bg!=0 → fill_rect（palette_color(bg)）；ch!=0 → text_glyph（palette_color(fg)）。
  - cursor block：`(cur_col, cur_row)` 处白色实心方块（8×16）。

## 决策

1. **codes[16] 数组（替逐 code 回调）**：38;5;N 是多 code 序列，回调式难前瞻。数组 + `i` 遍历 + `i+=2` 跳 5;N，干净。
2. **256 palette 算式（非 256 表）**：0-15 查 kAnsiPalette；16-231 6×6×6 立方算式；232-255 灰阶算式。省 1KB 表，算式纯整数。
3. **cursor block 白色实心**（非反色 cell）：简单可见，不依赖 cell 内容。always-on（无 blink，P6-c 简单）。
4. **paint per-cell bg 只画非默认**（bg!=0）：默认 cell（bg=0 黑）不 fill（整 terminal fill 已盖）。省 fill_rect。

## 陷阱

- **apply_sgr_ 前瞻边界**：`i+2 < ncode` 检查（防 38;5 越界）。codes[16] 上限（ncode<16）。
- **paint fill 数**：80×25 全 bg → 2000 fill_rect + 2000 glyph + cursor fill。PaintList 4096 容量边缘。但实际非默认 bg cell 少（ls 高亮少量），默认不 fill。OK。

## 验证

- 新 `test_terminal_bg256.cpp` 4 段：bg SGR（41/42 → bg 1/2，fg 仍白）+ 256 fg（38;5;200 → fg 200）+ 256 bg + reset（48;5;100 → bg 100，0m → bg 0）+ cursor block（flatten fills≥2 = bg+cursor）。
- standalone ctest **23/23** + ASAN 干净。
- Host ABI 零改动；core 仍 host-neutral。
