# 2026-06-30 · P0-b1 · PSF2 字体搬进本仓 + 解析器 + glyph 渲染单测

> P0-b 第一批（P0-b 拆 b1/b2/b3 的 b1）。把 CinuxOS 的 8×16 PSF2 字体作为 constexpr 资源搬进本仓 core/，配极简解析器，打通 `PsfFont.glyph()` → `swraster::glyph_blit()` 渲染链路。本批不画场景、不开 host（b2 干）。

## 背景

P0-b 要在 offscreen host 上用 swraster 画带文字的窗口（"Hello"）。前置：本仓得有字体 + 字体要能喂 `swraster::glyph_blit(bits, gw, gh, color, clip)`。

CinuxOS 的字体在 `assets/font.psf`（PSF2, 8×16, 256 glyphs, 16 bytes/glyph, MSB-first, 无 unicode table），渲染走 `.incbin` + `Framebuffer::put_pixel`。本仓无 Framebuffer，要用 Surface + glyph_blit。

## 目标

1. 字体进本仓 core/，**host-neutral**（不读文件、不依赖汇编）。
2. 极简 PSF2 解析器：parse header → `glyph(c)` 返 bitmap。
3. 证明 `glyph_blit` 能渲字（'A' 非空、' ' 空）。

## 设计

### 生成器 `scripts/gen_font_data.py`

把 `font.psf` 转成 `core/font_psf_data.hpp` 的 `constexpr uint8_t kFontPsfData[]`（+ `kFontPsfSize = sizeof(...)`）。**编译期常量**，不是运行时读文件——core 因此零 syscall、host-neutral。校验 PSF2 magic + 打印规格，4128 bytes → 369 行 hpp。

### `core/font.{hpp,cpp}` —— `PsfFont`

```cpp
class PsfFont {
public:
    void init();                       // parse kFontPsfData header
    const uint8_t* glyph(uint8_t c) const;  // 16 bytes, MSB-first; c>=256 -> glyph 0
    uint32_t width()/height()/num_glyphs()/bytes_per_glyph();
    bool ready();
};
```

**显式 little-endian byte 解析**（`le_u32`），不用 packed struct + reinterpret：
- 无 strict-aliasing UB；
- 不依赖 host endianness（虽 PC host 都是 LE，但显式 LE 更严格）；
- 符合 CODING-TASTE §13b「核过位布局」精神。

### 与 glyph_blit 的天然对接

PSF2 每 glyph = `height` bytes，每 byte 1 row，MSB-first，width=8。`swraster::glyph_blit` 内部 `bytes_per_row = (gw+7)/8 = 1`——**正好匹配**。渲染一行：
```cpp
swraster::glyph_blit(surf, x, y, font.glyph('H'), font.width(), font.height(), color, clip);
```

## 决策

- **C 数组 constexpr vs `.incbin`**：选 C 数组——纯 C++、零汇编、跨平台、host-neutral、CMake 不用特殊配。`.incbin` 要汇编 + objcopy，core host-neutral 上有 platform 风险。
- **显式 `le_u32` vs reinterpret packed struct**（CinuxOS font.cpp 的写法）：选显式 `le_u32`——无别名 UB + 跨 endianness 严格。多几行换确定性，值。
- **字体放 `core/`**：字体是 host-neutral 资源（任何 host 渲文字都用），`glyph_blit` 也在 core。放 `core/font.*` + `core/font_psf_data.hpp`。core 只多了 `constexpr` 数组 + 纯整数解析，不破「core 只 cstdint/cstddef」。

## 陷阱

- **PSF2 header 是 little-endian**：显式 `le_u32` 解析，不假设 host LE。
- **constexpr 数组 ≠ 读文件**：`kFontPsfData` 是编译期常量，core 不做 syscall，host-neutral 不破。
- **笔误教训**：测试循环里写成 `kH_font`（未定义），应是 `kFh`——构建前自查抓到。clangd 因 include 滞后没报这个（它在 `font.hpp not found` 后就停了），**靠的是真编译**。
- **clangd 滞后**（同 P0-a）：改 include 后 clangd 沿旧 compile_commands 解析报一片 false error，`cmake -B build` 重生成后自消。
- **`.clang-format` 仍缺**（pre-existing debt，P0-a 已记）：本批代码继续手动对齐现有风格。

## 验证

- **standalone ctest 全绿**（Release）：7 断言——原 6 项 + **PSF2 字体**（`ready()` / 8×16 / 256 glyphs / 'A' 渲前景像素 >0 / ' ' 渲 0 像素）。
- **ASAN 干净**：`-fsanitize=address` 构建 + ctest + 直接运行均 exit 0。
